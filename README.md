# Minimal E-commerce 🚀

A high-performance, professional-grade E-commerce application built with **Next.js 16** and **TypeScript**. This project demonstrates advanced frontend architecture, state management, and a robust CI/CD pipeline.

[\[Live Demo Link Here\]](https://minimal-mart.vercel.app)
[\[Portfolio Link Here\]](www.thureinphyoko.com)

## 🌟 Key Features
- **Authentication & Security:** Protected routes for the Dashboard using a simulated Auth flow.
- **Complex Commerce Flow:** Complete lifecycle from Wishlist → Cart → Checkout → Order confirmation.
- **Advanced Data Handling:** Server-side pagination, multi-criteria filtering, and sorting using **TanStack Query**.
- **Management Dashboard:** Full CRUD functionality to manage products and view analytics.
- **Smooth UX:** Optimized with `use-debounce` for searches and **Framer Motion** for micro-interactions.

## 🛠 Tech Stack
- **Framework:** Next.js 16 (App Router)
- **State Management:** Zustand (Persistent Cart & Wishlist)
- **Data Fetching:** TanStack Query (React Query) & Axios
- **Styling:** Tailwind CSS 4.0 & Material UI (MUI)
- **Animations:** Framer Motion
- **Icons/UI:** Lucide React & Sonner (Toasts)
- **Testing** Jest and React Testing Library

## 🏗 Engineering & DevOps (The "Senior" Focus)
This project is built with a focus on code reliability and professional team workflows:

- **Automated Testing:** Unit and integration tests using **Jest** and **React Testing Library**.
- **CI/CD Pipeline:** Integrated **GitHub Actions** that automatically runs the test suite on every Push and Pull Request.
- **Branch Protection:** Production-grade rulesets prevent merging code to `main` unless all CI checks pass.
- **Deployment:** Continuous Deployment via **Vercel**, synced with GitHub status checks.

## 🚀 Getting Started
1. Clone the repo: `git clone https://github.com/ThuReinPhyoKo/Minimal-E-commerce`
2. Install dependencies: `npm install`
3. Run the development server: `npm run dev`
4. Run tests: `npm run test`
